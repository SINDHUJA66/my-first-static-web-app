Web Font Delivery

There are a few different methods for delivering fonts to the client browser when building a website. One approach is to use web fonts, which allow fonts to be loaded from a server when a page is requested.

To use a web font, it first needs to be converted into a web-friendly format like WOFF or WOFF2. Then the font files can be hosted on a web server just like any other asset. In the CSS for the site, the web font can be specified using @font-face which defines the font files to load.

When the browser makes a request for the page, the web font files will be downloaded along with the rest of the site assets. The browser will have the fonts available to render text with the desired typography. Web fonts allow using any font on the web, not just default system fonts. This is a simple method for delivering a custom font experience. It does increase page load size, so optimization techniques like font subsetting should be used. But overall, web fonts give great control over typography for branding and design on the web.
